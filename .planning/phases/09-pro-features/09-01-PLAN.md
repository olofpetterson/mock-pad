  <name>Task 1: Extend ProManager with StoreKit 2 product fetch, purchase, restore, and transaction monitoring</name>
  <files>MockPad/MockPad/App/ProManager.swift</files>
  <action>
Add `import StoreKit` to ProManager.swift.

Add a static product ID constant: `static let productID = "com.olof.petterson.MockPad.pro"`

Add published properties to ProManager:
- `private(set) var product: Product?` -- fetched StoreKit product
- `private(set) var purchaseState: PurchaseState = .idle` -- current purchase flow state

Add PurchaseState enum inside ProManager:
```
enum PurchaseState: Equatable {
    case idle
    case purchasing
    case purchased
    case failed(String)
    case pending
}
```
Make it Equatable so views can compare states.

Add a private `transactionListener: Task<Void, Never>?` property.

In `init()`, after loading isPro from Keychain, start the transaction listener:
```
transactionListener = Task {
    for await result in Transaction.updates {
        await handleTransactionUpdate(result)
    }
}
```

Add these methods to ProManager:

1. `func loadProduct() async` -- fetches product via `Product.products(for: [Self.productID])`, sets `product = products.first`. Guard early if product is already loaded. Catch errors silently (product stays nil, paywall shows loading state).

2. `func purchase() async` -- guards that product is non-nil. Sets purchaseState to .purchasing. Calls `product.purchase()`. Handles result:
   - `.success(let verification)`: Extract `verification.payloadValue` (try? to skip failed verification). Call `setPro(true)`, set `purchaseState = .purchased`, call `await transaction.finish()`.
   - `.userCancelled`: Set `purchaseState = .idle`.
   - `.pending`: Set `purchaseState = .pending`.
   - `@unknown default`: Set `purchaseState = .idle`.
   - Catch block: Set `purchaseState = .failed(error.localizedDescription)`.

3. `func restorePurchases() async` -- iterates `Transaction.currentEntitlements`. For each, extract payloadValue (try?), check `transaction.productID == Self.productID` and `transaction.revocationDate == nil`. If found, call `setPro(true)` and return. If loop completes without finding a valid entitlement, set `purchaseState = .failed("No previous purchase found.")`.

4. `func checkEntitlements() async` -- iterates `Transaction.currentEntitlements`. If valid entitlement found (matching productID, no revocationDate), call `setPro(true)` and return. If loop completes without match AND isPro is currently true, call `setPro(false)` (handles refunds).

5. `private func handleTransactionUpdate(_ result: VerificationResult<Transaction>) async` -- extract payloadValue (try?), check productID matches. If revocationDate is nil, call `setPro(true)`. If revocationDate is non-nil, call `setPro(false)`. Always call `await transaction.finish()`.

Keep all existing methods (setPro, canAddEndpoint, canImportEndpoints) unchanged.

IMPORTANT: Since SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor, ProManager is implicitly MainActor-isolated. StoreKit 2 async calls resume on MainActor after await, so property mutations are safe. Do NOT add explicit @MainActor annotations or actor isolation -- the default handles it.

IMPORTANT: Never hardcode "$5.99" anywhere. Always use `product.displayPrice` (this is consumed in Plan 02's paywall view, not here).

```json

  "type" : "com.apple.product-type.storekit",
  "version" : 1,
  "products" : [
    {
      "displayPrice" : "5.99",
      "familyShareable" : false,
      "internalID" : "1",
      "localizations" : [
        {
          "description" : "Unlock unlimited endpoints, OpenAPI import, custom templates, collections, response delay, and export/share.",
          "displayName" : "MockPad PRO",
          "locale" : "en_US"
        }
      ],
      "productID" : "com.olof.petterson.MockPad.pro",
      "referenceName" : "MockPad PRO",
      "type" : "NonConsumable"
    }
  ],
  "settings" : {
    "_failTransactionsEnabled" : false,
    "_locale" : "en_US",
    "_storefront" : "USA",
    "_storeKitErrors" : []
  }
}
```

Note: The user will need to manually select this file in Xcode scheme settings (Edit Scheme > Run > Options > StoreKit Configuration) for local testing. This is a manual Xcode step that cannot be automated via code.

In ContentView.swift, add product pre-fetching and entitlement checking:
1. Add `@Environment(ProManager.self) private var proManager` (it is already injected via .environment() in MockPadApp).

   ```
   await proManager.loadProduct()
   await proManager.checkEntitlements()
   ```
   This runs once at app launch, asynchronously, without blocking UI. The .task modifier handles cancellation automatically.

The .task modifier should be placed after the existing .onChange(of: scenePhase) modifier.
