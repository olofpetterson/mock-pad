  <behavior>
    **EndpointMatcher.matchPath(pattern:against:) -> [String: String]?**
    - Splits both pattern and path by "/" using .split(separator:) (handles leading/trailing slashes)
    - Literal segments: case-insensitive exact match
    - `:param` segments: match any single segment, capture value with param name as key
    - `*` at end of pattern: match any remaining segments (0+)
    - Returns nil for no match, [:] for exact match, ["id": "42"] for param match

    Cases:
    - matchPath("/api/users", "/api/users") -> [:] (exact)
    - matchPath("/api/users/:id", "/api/users/42") -> ["id": "42"]
    - matchPath("/api/users/:userId/posts/:postId", "/api/users/42/posts/7") -> ["userId": "42", "postId": "7"]
    - matchPath("/api/*", "/api/users/42/posts") -> [:] (wildcard, no params before *)
    - matchPath("/api/:version/*", "/api/v2/users") -> ["version": "v2"]
    - matchPath("/api/users/:id", "/api/users") -> nil (segment count mismatch)
    - matchPath("/api/users", "/api/users/42") -> nil (too many segments)
    - matchPath("/api/*", "/api") -> [:] (wildcard matches zero sub-segments)

    **EndpointMatcher.specificity(of:) -> Int**
    - 0 = exact (no : or * segments)
    - 1 = parameterized (has : segments, no *)
    - 2 = wildcard (ends with *)

    **EndpointMatcher.match() updated behavior:**
    - Phase 1: collect ALL path matches (using matchPath) with extracted params
    - Phase 2: sort by specificity (exact > param > wildcard)
    - Phase 3: find first method match in sorted list
    - If no method match: 405 with allowed methods from ALL path-matched endpoints

    **MatchResult.matched gains pathParams: [String: String] as 7th associated value**
    - Exact matches: pathParams = [:]
    - Parameterized matches: pathParams = ["id": "42", ...]
    - Wildcard matches: pathParams = [:] (or params from segments before *)

    **PathParamReplacer.replace(in:with:) -> String**
    - Replaces {key} tokens with values from params dictionary
    - Empty params: returns body unchanged (early return guard)
    - No matching tokens: returns body unchanged
    - Multiple tokens: all replaced
    - Case-sensitive: {id} != {ID}

    Cases:
    - replace('{"id": {id}}', ["id": "42"]) -> '{"id": 42}'
    - replace('{"name": "User {id}"}', ["id": "42"]) -> '{"name": "User 42"}'
    - replace('{"userId": {userId}, "postId": {postId}}', ["userId": "42", "postId": "7"]) -> '{"userId": 42, "postId": 7}'
    - replace('{"id": {id}}', [:]) -> '{"id": {id}}' (no params, unchanged)
    - replace('no tokens', ["id": "42"]) -> 'no tokens' (no matching tokens)
    - replace('{ID}', ["id": "42"]) -> '{ID}' (case-sensitive, no match)
  </behavior>
