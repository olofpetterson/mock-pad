**AddEndpointSheet.swift:** Build a sheet for creating new endpoints:
- `@Environment(EndpointStore.self)` for adding
- `@Environment(ServerStore.self)` for engine sync
- `@Environment(\.dismiss)` for sheet dismissal
- Local @State fields: `path: String = "/api/"`, `httpMethod: String = "GET"`, `responseStatusCode: Int = 200`
- Form layout: Section for "Path" with TextField (monospaced, placeholder "/api/resource"), Section for "Method" with HTTPMethodPickerView, Section for "Status Code" with StatusCodePickerView.
- Toolbar: Cancel button (dismiss), "Create" button that:
  1. Creates `MockEndpoint(path: path, httpMethod: httpMethod, responseStatusCode: responseStatusCode, sortOrder: endpointStore.endpointCount)`
  2. Calls `endpointStore.addEndpoint(endpoint)`
  3. Calls `Task { await serverStore.updateEngineEndpoints(endpointStore: endpointStore) }`
  4. Dismisses the sheet
- Create button disabled if path is empty or whitespace-only.
- NavigationStack wrapper with title "New Endpoint".
- Background: `MockPadColors.background`. Section backgrounds: `MockPadColors.panel`.

**EndpointEditorView.swift:** Build the full endpoint editor:
- `@Bindable var endpoint: MockEndpoint` for two-way binding to all fields.
- `@Environment(EndpointStore.self)` for save.
- `@Environment(ServerStore.self)` for engine sync.
- `@State private var syncTask: Task<Void, Never>?` for debounced sync.

Form layout with sections:
1. **Path section:** TextField bound to `$endpoint.path`, monospaced font (`MockPadTypography.monoInput`), placeholder "/api/resource".
2. **Method section:** HTTPMethodPickerView bound to `$endpoint.httpMethod`.
3. **Status Code section:** StatusCodePickerView bound to `$endpoint.responseStatusCode`.
4. **Response Body section:** Placeholder Text "Response body editor (Plan 03)" -- the actual ResponseBodyEditorView is created in Plan 03 and will be wired here.
5. **Response Headers section:** Placeholder Text "Response headers editor (Plan 03)" -- the actual ResponseHeadersEditorView is created in Plan 03 and will be wired here.

Auto-save: Use `.onChange(of: endpoint.path)`, `.onChange(of: endpoint.httpMethod)`, `.onChange(of: endpoint.responseStatusCode)` to call `saveAndSync()`. The `saveAndSync()` method calls `endpointStore.updateEndpoint()` immediately (SwiftData save), then `debouncedSyncEngine()` (debounced engine update at 300ms).

Navigation title: endpoint.path (dynamic). Background: `MockPadColors.background`.
