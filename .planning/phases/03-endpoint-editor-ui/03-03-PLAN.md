  <action>
Build a SwiftUI view for editing the JSON response body with live validation.

**Parameters:**
- `@Binding var text: String` -- bound to endpoint.responseBody
- `let onChanged: () -> Void` -- callback for save+sync after edits

**JSON Validation Logic (private computed property or method):**
Define a private enum `JSONValidationResult` with cases: `.valid`, `.invalid(String)`, `.empty`. Implement `validateJSON(_ text: String) -> JSONValidationResult`:
- If text trimmed is empty, return `.empty`
- If `text.data(using: .utf8)` fails, return `.invalid("Invalid UTF-8 encoding")`
- Try `JSONSerialization.jsonObject(with: data)` -- if succeeds return `.valid`, if fails return `.invalid(error.localizedDescription)`

**Pretty-Print Logic (private method):**
Implement `prettyPrintJSON(_ text: String) -> String?`:
- Parse with `JSONSerialization.jsonObject(with:)`
- Re-encode with `JSONSerialization.data(withJSONObject:options: [.prettyPrinted, .sortedKeys])`
- Return the String, or nil if input is not valid JSON

**Layout:**
- VStack with:
  1. HStack toolbar: "Response Body" section label (`.mockSectionTitleStyle()`), Spacer, "Format" button (disabled when JSON is invalid or empty -- calls prettyPrintJSON and assigns result to text binding, then calls onChanged), JSON validation badge
  2. TextEditor bound to `$text`

**JSON Validation Badge:** HStack with SF Symbol + text:
- Valid: `checkmark.circle.fill` green (`MockPadColors.status2xx`) + "Valid JSON"
- Invalid: `xmark.circle.fill` red (`MockPadColors.status5xx`) + "Invalid JSON"
- Empty: no badge shown (hide the HStack)
- Font: `.system(.caption, design: .monospaced)`

**TextEditor styling:**
- Font: `MockPadTypography.codeEditor` (monospaced callout)
- Minimum height: `MockPadMetrics.editorMinHeight` (200pt)
- Frame with minHeight, maxHeight `.infinity`
- Background: `MockPadColors.surface`
- Foreground: `MockPadColors.textPrimary`
- Border: 1px `MockPadColors.border`, cornerRadius `MockPadMetrics.cornerRadiusSmall`
- Padding: `MockPadMetrics.editorPadding`
- `.scrollContentBackground(.hidden)` to remove default TextEditor background

**On text change:** Use `.onChange(of: text)` to call `onChanged()` -- the parent handles save and sync.

**Format button:** Text "Format" or SF Symbol `arrow.trianglehead.branch` / `text.alignleft`, styled with `MockPadTypography.buttonSmall`, `MockPadColors.accent` foreground. Disabled (opacity 0.4) when validateJSON returns non-valid.

**ResponseHeadersEditorView.swift:** Build a key-value pair editor for response headers.

**Parameters:**
- `let endpoint: MockEndpoint` -- direct reference for reading/writing responseHeaders
- `let onChanged: () -> Void` -- callback for save+sync

**Local state:**
- `@State private var headerPairs: [(id: UUID, key: String, value: String)] = []`
  Use UUID for stable ForEach identity since tuples are not Identifiable.

**Initialization (.onAppear or .task):**
- Load `endpoint.responseHeaders` sorted by key alphabetically
- Map to `headerPairs` array with UUID for each pair

**Layout:**
- VStack with section header "Response Headers" (`.mockSectionTitleStyle()`) and "Add Header" button
- ForEach over `$headerPairs` (using \.id): each row is an HStack with:
  - TextField "Key" bound to the pair's key (`.font(MockPadTypography.monoSmall)`)
  - TextField "Value" bound to the pair's value (`.font(MockPadTypography.monoSmall)`)
  - Button with `minus.circle.fill` SF Symbol (red) to remove the pair
- "Add Header" button: Appends `(id: UUID(), key: "", value: "")` to headerPairs

**Saving headers back to endpoint:**
- On any header field change (`.onChange` on the headerPairs array or on individual fields using `.onSubmit` / focus loss):
  Implement `saveHeaders()` method that:
  1. Filters out pairs with empty keys
  2. Builds `[String: String]` dictionary from remaining pairs
  3. Assigns to `endpoint.responseHeaders` (uses the computed property setter which encodes to JSON Data)
  4. Calls `onChanged()`

- Since observing individual tuple field changes is complex, use `.onChange(of: headerPairs.map { "\($0.key):\($0.value)" })` to detect changes (join key-value as strings for Equatable comparison). Call `saveHeaders()` on change.

**Styling:**
- TextField backgrounds: `MockPadColors.surface`, border `MockPadColors.border`, cornerRadius `MockPadMetrics.cornerRadiusSmall`
- Add button: accent color, `plus.circle.fill` icon
- Delete button: `MockPadColors.status5xx` (red), `minus.circle.fill` icon
- Row spacing: `MockPadMetrics.rowSpacing`

**EndpointEditorView.swift updates:**
Replace the two placeholder Text views with the actual editor components:

1. Response Body section: Replace placeholder with `ResponseBodyEditorView(text: $endpoint.responseBody, onChanged: { saveAndSync() })`. This uses @Bindable's `$endpoint.responseBody` binding directly.

2. Response Headers section: Replace placeholder with `ResponseHeadersEditorView(endpoint: endpoint, onChanged: { saveAndSync() })`.

Add `.onChange(of: endpoint.responseBody)` to the existing onChange handlers if not already covered by ResponseBodyEditorView's onChanged callback. The onChanged callback should be sufficient -- do NOT double-trigger.

Ensure `saveAndSync()` method calls `endpointStore.updateEndpoint()` then `debouncedSyncEngine()` (already implemented in Plan 02).
