**EndpointRowView.swift:** Create `Views/` directory under `MockPad/MockPad/`. Build a SwiftUI view that displays a single endpoint row:
- Accept `endpoint: MockEndpoint` parameter (NOT @Bindable -- read-only display)
- Accept `onToggle: (Bool) -> Void` closure for enable/disable toggle
- Layout: HStack with method badge (colored pill using `MockPadColors.methodColor(for:)` and `MockPadTypography.methodBadge`), path text (`.endpointPathStyle()`), spacer, status code text (colored by `MockPadColors.statusCodeColor(code:)`), and Toggle (labelsHidden, calls onToggle)
- Method badge: Text(endpoint.httpMethod) with `.methodBadgeStyle(color:)` extension, fixed width `MockPadMetrics.methodBadgeWidth`
- Disabled endpoints: reduce row opacity to 0.5
- Background: `MockPadColors.panel` with `MockPadMetrics.cornerRadius` corner radius
- Minimum row height: `MockPadMetrics.endpointCardHeight`

**EndpointListView.swift:** Build the main list view:
- `@Environment(EndpointStore.self)` for endpoint data
- `@Environment(ServerStore.self)` for engine sync
- `@Environment(ProManager.self)` for add-endpoint gating
- `@State private var showAddSheet = false` for new endpoint sheet (the sheet itself is built in Plan 02)
- `@State private var syncTask: Task<Void, Never>?` for debounced engine sync

NavigationStack wrapping a List of `ForEach(endpointStore.endpoints)` with EndpointRowView.

For each row, provide swipe actions:
- Leading swipe: Duplicate button (square.on.square icon, tint accent). On tap, create new MockEndpoint with same fields as source, sortOrder = max + 1, call `endpointStore.addEndpoint(copy)`, then `debouncedSyncEngine()`.
- Trailing swipe: Delete button (trash icon, tint destructive). On tap, call `endpointStore.deleteEndpoint(endpoint)`, then `debouncedSyncEngine()`.

Enable/disable toggle in EndpointRowView: onToggle closure sets `endpoint.isEnabled = newValue`, calls `endpointStore.updateEndpoint()`, then `debouncedSyncEngine()`.

Reorder: `.onMove(perform: moveEndpoints)`. The `moveEndpoints(from:to:)` method: copy `endpointStore.endpoints` to local array, call `.move(fromOffsets:toOffset:)`, reassign `endpoint.sortOrder = index` for each, call `endpointStore.updateEndpoint()`, then `debouncedSyncEngine()`.

Toolbar: `.toolbar { EditButton() }` for reorder mode toggle. Add a "+" button that checks `proManager.canAddEndpoint(currentCount: endpointStore.endpointCount)` -- if allowed, sets `showAddSheet = true`; if not, shows a basic alert that PRO is required (full paywall is Phase 9). Attach `.sheet(isPresented: $showAddSheet)` with placeholder text "Add Endpoint" (the real sheet content is built in Plan 02).

`debouncedSyncEngine()`: Cancel `syncTask`, create new Task with 300ms sleep then `await serverStore.updateEngineEndpoints(endpointStore: endpointStore)`.

NavigationStack title: "Endpoints".
Background: `MockPadColors.background`.
List style: `.listStyle(.plain)`, list row background: `MockPadColors.background`.

**ContentView.swift:** Replace the placeholder `VStack { Text("MockPad") }` body with `EndpointListView()`. Keep the existing `.onChange(of: scenePhase)` lifecycle handling intact. The scenePhase handler must remain on ContentView (it has the @Environment stores).
