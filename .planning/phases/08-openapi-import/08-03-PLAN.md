struct OpenAPIPreviewSheet: View {

**Menu modification:**
In the existing toolbar Menu (the `ellipsis.circle` one), add a new item BELOW the existing "Import from File" button, separated by another Divider:

```swift
Divider()

Button("Import OpenAPI Spec", systemImage: "doc.badge.gearshape") {

    }
    showOpenAPIImporter = true
}
```

The menu structure becomes:
1. Export to File (PRO)
2. Share Collection (PRO)
3. Divider
4. Import from File
5. Divider
6. Import OpenAPI Spec (PRO)

**New fileImporter modifier** (add after existing `.fileImporter`):
```swift
.fileImporter(
    isPresented: $showOpenAPIImporter,
    allowedContentTypes: [
        .json,
        UTType(filenameExtension: "yaml") ?? .plainText,
        UTType(filenameExtension: "yml") ?? .plainText
    ],
    allowsMultipleSelection: false
) { result in
    guard let url = (try? result.get())?.first else { return }
    let accessing = url.startAccessingSecurityScopedResource()
    defer { if accessing { url.stopAccessingSecurityScopedResource() } }
    do {
        let data = try Data(contentsOf: url)
        let parseResult = try OpenAPIParser.parse(data: data)
        pendingOpenAPIImport = parseResult
        showOpenAPIPreview = true
    } catch {
        importError = error.localizedDescription
        showImportError = true
    }
}
```

**New sheet modifier** (add after existing `.sheet(isPresented: $showImportPreview)`):
```swift
.sheet(isPresented: $showOpenAPIPreview, onDismiss: {
    pendingOpenAPIImport = nil
    debouncedSyncEngine()
}) {
    if let parseResult = pendingOpenAPIImport {
        OpenAPIPreviewSheet(parseResult: parseResult)
    }
}
```

**New alert for OpenAPI PRO gating** (add after existing alerts):
```swift
.alert("PRO Required", isPresented: $showOpenAPIProAlert) {
    Button("OK", role: .cancel) {}
} message: {
    Text("Upgrade to PRO to import OpenAPI specifications.")
}
```

**Reuse existing `importError`/`showImportError` state** for OpenAPI parse errors -- the error alert is generic enough.

**Note:** The security-scoped file access pattern (`startAccessingSecurityScopedResource`/`stopAccessingSecurityScopedResource` with `defer`) matches the existing MockPad JSON import pattern exactly.
