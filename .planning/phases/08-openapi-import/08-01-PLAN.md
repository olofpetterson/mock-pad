Build `YAMLConverter` as a caseless enum service (matches CollectionImporter, CurlGenerator pattern) with a single public API:

```swift
enum YAMLConverter {
    enum ConversionError: Error, LocalizedError, Equatable {
        case invalidYAML(String)
        var errorDescription: String? { ... }
    }

    /// Convert YAML string to JSON-compatible [String: Any] or [Any]
    static func toJSON(_ yaml: String) throws -> Data
}
```

**Implementation approach (line-by-line parser):**
1. Split input by newlines. Track indentation with a stack.
2. For each line: calculate indent level (count leading spaces), strip inline comments (# outside quotes), detect array items (`- `), detect key-value pairs (`key: value`), detect bare scalars.
3. Build nested `[String: Any]` / `[Any]` structure based on indentation changes:
   - Indent increase = nested object or array starts
   - Indent decrease = pop back to parent level
   - Same indent = sibling entry
4. Scalar parsing (`parseScalar`): quoted strings (single/double) -> String, `true`/`false`/`True`/`False` -> Bool, `null`/`~`/`Null` -> NSNull(), integers -> Int, floats (with `.`) -> Double, flow collections (starts with `[` or `{`) -> parse via `JSONSerialization`, unquoted strings -> String.
5. Flow collections: when a value starts with `[` or `{`, pass to `JSONSerialization.jsonObject(with:)` for parsing. This handles `required: [id, name]` and `enum: [active, inactive]` patterns common in OpenAPI YAML.
6. Handle multiline: if a value ends with `|` or `>`, collect subsequent indented lines as a single string value (join with newlines for `|`, spaces for `>`). This is needed for description fields in OpenAPI.
7. Convert final parsed structure to JSON Data via `JSONSerialization.data(withJSONObject:)`.

**TDD RED-GREEN-REFACTOR cycle. Write tests FIRST, then implement.**

**Test cases (~15 tests):**
1. Simple key-value: `name: Petstore` -> `{"name": "Petstore"}`
2. Nested mapping (2 levels): `info:\n  title: API\n  version: "1.0"` -> correct nesting
3. Nested mapping (3+ levels): deep nesting with different indent widths
4. Block sequence: `- item1\n- item2` -> `["item1", "item2"]`
5. Sequence of mappings: `- name: a\n  value: 1\n- name: b\n  value: 2` -> array of objects
6. Scalar types: integers, floats, booleans (true/false/True/False), null/~/Null
7. Quoted strings (double quotes): preserve content, handle colon in value
8. Quoted strings (single quotes): preserve content
9. Flow collection array: `required: [id, name, email]` -> array value
10. Flow collection object: `example: {"id": 1, "name": "test"}` -> object value
11. Inline comments: `port: 8080 # server port` -> strips comment
12. Empty values (nested object follows): `paths:` with no value -> key with nested children
13. Mixed content: realistic OpenAPI snippet with paths, operations, responses
14. Multiline literal block (`|`): collects indented lines as joined string
15. Error case: truly invalid YAML (mismatched indentation causing ambiguity) throws ConversionError
