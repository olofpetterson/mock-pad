**ServerConfiguration** (MockPad/MockPad/Models/ServerConfiguration.swift):

Add a new `localhostOnly` static property following the exact same pattern as `corsEnabled` and `autoStart` -- defensive `object(forKey:) == nil` check defaulting to `true`:

```swift
static var localhostOnly: Bool {
    get {
        UserDefaults.standard.object(forKey: "localhostOnly") == nil
            ? true
            : UserDefaults.standard.bool(forKey: "localhostOnly")
    }
    set {
        UserDefaults.standard.set(newValue, forKey: "localhostOnly")
    }
}
```

Default is `true` (security-conscious: only localhost by default).

**MockServerEngine** (MockPad/MockPad/Services/MockServerEngine.swift):

1. Update `start()` signature to accept `localhostOnly: Bool`:
```swift
func start(port: UInt16, endpoints: [EndpointSnapshot], corsEnabled: Bool, localhostOnly: Bool) throws
```

2. Store the localhostOnly value in a private property: `private var localhostOnly: Bool = true`

3. After creating the NWListener, conditionally set `requiredLocalEndpoint`:
```swift
if localhostOnly {
    let params = listener.parameters
    // The listener already has TCP parameters set up
    // To bind to loopback only, use service with specific host
}
```

Actually, the NWListener binding approach: The cleanest way is to create the listener differently depending on localhostOnly. Currently the engine likely creates `NWListener(using: tcpParams)` and then sets the port. For localhost-only binding:

- If localhostOnly is true: Create listener normally but set `listener.service = nil` and instead use a specific endpoint. However, NWListener's API for binding to a specific interface is limited. The practical approach is:
  - Create `NWListener(using: params)` with the port as before
  - The NWListener already listens on the specified port; iOS apps running on-device inherently only have localhost accessible unless on same network
  - For true loopback binding, check if the current engine code already uses `NWEndpoint.hostPort`. If it creates the listener with just a port number via `NWParameters.requiredLocalEndpoint`, use `.hostPort(host: .ipv4(.loopback), port: NWEndpoint.Port(integerLiteral: port))`

Read the current start() implementation carefully. Find where NWListener is created and the port is configured. Add conditional logic:

```swift
if localhostOnly {
    let endpoint = NWEndpoint.hostPort(
        host: NWEndpoint.Host.ipv4(.loopback),
        port: NWEndpoint.Port(integerLiteral: port)
    )
    tcpParameters.requiredLocalEndpoint = endpoint
}
```

This must be applied to the TCP parameters BEFORE creating the NWListener. The `requiredLocalEndpoint` on NWParameters restricts which local address the listener binds to.

Store `self.localhostOnly = localhostOnly` early in the start method for reference.

Also register "localhostOnly" in the UserDefaults.standard.register(defaults:) call in MockPadApp.swift init -- add `"localhostOnly": true` to the defaults dictionary.
