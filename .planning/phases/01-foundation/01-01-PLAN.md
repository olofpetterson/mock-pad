  <action>
Create the `Utilities/` directory under `MockPad/MockPad/`.

**ServerConfiguration.swift** - An enum (caseless, used as namespace) with static computed properties backed by UserDefaults:
- `static var port: UInt16` - get reads "serverPort" from UserDefaults, returns UInt16(clamping:) of the Int value (default 8080 if value is 0). Set writes Int(newValue) to "serverPort".
- `static var corsEnabled: Bool` - get/set for "corsEnabled" key. Default true (use `register(defaults:)` pattern -- the app entry point will call `UserDefaults.standard.register(defaults:)` in Plan 02, so this property can use simple `.bool(forKey:)` after defaults are registered).
- `static var autoStart: Bool` - get/set for "autoStart" key. Same pattern.

For the Bool properties, since UserDefaults.standard.register(defaults:) will be called at app launch (Plan 02), simple `UserDefaults.standard.bool(forKey:)` is safe for the getter. However, to be defensive (in case accessed before register), check `object(forKey:) == nil` and return `true` as fallback.

Import `Foundation`.

**KeychainService.swift** - A caseless enum wrapping Apple Security framework:
- `private static let service = "com.olof.petterson.MockPad"`
- `static func saveBool(_ value: Bool, forKey key: String)` - encodes Bool as single-byte Data ([1] for true, [0] for false). Builds query dict with kSecClass/kSecAttrService/kSecAttrAccount/kSecValueData. Calls SecItemDelete first (to avoid errSecDuplicateItem), then SecItemAdd. Discard return status.
- `static func loadBool(forKey key: String) -> Bool?` - builds query dict with kSecReturnData + kSecMatchLimitOne. Calls SecItemCopyMatching. Returns nil if status != errSecSuccess. Otherwise casts result to Data and returns data[0] == 1.
- `static func delete(forKey key: String)` - builds query dict, calls SecItemDelete. Discard return status.

Import `Foundation` and `Security`.
  </action>

  </verify>
  <done>
ServerConfiguration provides UserDefaults-backed server settings with correct defaults (port 8080, CORS on, auto-start on). KeychainService provides zero-dependency Keychain wrapper for Bool persistence using Security framework.
  </done>
