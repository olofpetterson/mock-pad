  <name>Task 1: Add VoiceOver and tap targets to EndpointRowView, ServerStatusBarView, and RequestLogRowView</name>
  <files>MockPad/MockPad/Views/EndpointRowView.swift, MockPad/MockPad/Views/ServerStatusBarView.swift, MockPad/MockPad/Views/RequestLogRowView.swift</files>
  <action>
**EndpointRowView.swift:**
Use `.accessibilityElement(children: .contain)` on the outer HStack so the toggle remains independently activatable by VoiceOver (do NOT use `.combine` -- it swallows the toggle's adjustable trait per research pitfall #1).

Add to the outer HStack (after `.opacity()`):
```swift
.accessibilityElement(children: .contain)
.accessibilityLabel("\(endpoint.httpMethod) \(endpoint.path), status \(endpoint.responseStatusCode)")
```

Add to the Toggle (after `.tint()`):
```swift
.accessibilityLabel("\(endpoint.httpMethod) endpoint enabled")
.accessibilityValue(endpoint.isEnabled ? "On" : "Off")
```

Hide individual text elements that are covered by the row-level label -- add `.accessibilityHidden(true)` to: the method badge Text, the path Text, and the status code Text.

**ServerStatusBarView.swift:**
Hide the decorative status Circle: add `.accessibilityHidden(true)` after the `.frame()`.

Group the server status VStack with `.accessibilityElement(children: .combine)`.

Add to the Button (after the closing `} label:` block):
```swift
.accessibilityLabel(serverStore.isRunning ? "Stop server" : "Start server")
.accessibilityHint("Double tap to \(serverStore.isRunning ? "stop" : "start") the mock server")
.frame(minWidth: 44, minHeight: 44)
```
Add `.contentShape(Rectangle())` before the button's label closure to ensure the expanded 44pt frame is fully tappable.

**RequestLogRowView.swift:**
Add to the outer HStack (after `.frame(height:)`):
```swift
.accessibilityElement(children: .combine)
.accessibilityLabel("\(log.method) \(log.path), status \(log.responseStatusCode), \(String(format: "%.0f", log.responseTimeMs)) milliseconds")
```
  </action>

  <done>EndpointRowView has `.contain` grouping with toggle independently accessible, ServerStatusBarView has descriptive labels and 44pt button target, RequestLogRowView has composite VoiceOver label.</done>

**LogFilterChipsView.swift:**
In the `chipButton` function, add to the Button (after `.buttonStyle(.plain)`):

```swift
.accessibilityAddTraits(isActive ? .isSelected : [])
.accessibilityRemoveTraits(isActive ? [] : .isSelected)
```

**CollectionFilterChipsView.swift:**

```swift
.accessibilityLabel("\(label) collection filter")

```
Change the chip's vertical padding from `.padding(.vertical, 6)` to `.frame(minHeight: MockPadMetrics.minTouchHeight)` to ensure 44pt minimum tap height. Keep the `.padding(.horizontal, 12)`.

**HTTPMethodPickerView.swift:**
Add to each Button (after `.buttonStyle(.plain)`):
```swift
.accessibilityLabel("\(method) method")
.accessibilityAddTraits(isSelected ? .isSelected : [])
.accessibilityRemoveTraits(isSelected ? [] : .isSelected)
```
Note: `isSelected` is already computed inside the label closure. Move `let isSelected = selectedMethod == method` outside the label closure (before the Button) so it's accessible for the accessibility modifiers.

Change `.frame(maxWidth: .infinity, minHeight: 36)` to `.frame(maxWidth: .infinity, minHeight: MockPadMetrics.minTouchHeight)` (44pt).

Change `.frame(minWidth: 56, minHeight: 36)` to `.frame(minWidth: 56, minHeight: MockPadMetrics.minTouchHeight)` (44pt).
